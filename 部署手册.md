# Astral / EasyTier 公网节点部署手册

> 本手册记录的是我这次在 `47.119.136.200` 上实际完成的部署方法，目标是把一台 Linux 服务器部署成一个可长期运行的 EasyTier 公网入口节点，使 Astral 客户端可以通过 `IP:端口` 连接到它。
>
> 本次实际部署结果：
> - 服务器：`47.119.136.200`
> - 系统：`Ubuntu 24.04 x86_64`
> - 服务名：`easytier-public.service`
> - 监听端口：`11010`
> - 监听协议：`TCP` + `UDP`
> - 二进制版本：`EasyTier v2.4.5`

---

## 1. 部署目标

部署完成后，你将得到一台长期在线的公网节点，具备以下特征：

- 对外监听 `tcp://0.0.0.0:11010`
- 对外监听 `udp://0.0.0.0:11010`
- 由 `systemd` 托管，服务器重启后自动拉起
- Astral 客户端可将该地址填写为服务器，例如：`47.119.136.200:11010`

这个节点本质上是一个 EasyTier 公网入口/中继节点，**不是 Astral 图形界面程序本体**。

---

## 2. 适用场景

适用于以下情况：

- 你已经有一台公网 Linux 服务器
- 你想让 Astral 使用你自己的服务器，而不是默认公共节点
- 你只需要先把“服务器入口节点”跑起来，不打算在服务器上跑 Flutter UI

推荐环境：

- `Ubuntu 22.04 / 24.04`
- `x86_64`
- 有公网 IP
- 已开放目标端口（本手册使用 `11010`）
- 具备 `root` 权限或可 `sudo`

---

## 3. 部署前准备

### Step 1：确认服务器基础条件

在服务器上执行：

```bash
uname -a
uname -m
cat /etc/os-release
```

你需要重点确认：

- 架构是 `x86_64` / `amd64`
- 系统是 Linux，最好是 Ubuntu

### Step 2：确认常用工具可用

执行：

```bash
command -v curl
command -v systemctl
command -v unzip
```

如果 `unzip` 不存在，后面会安装。

### Step 3：确认目标端口没有被占用

执行：

```bash
ss -lntup | grep 11010 || true
```

如果没有输出，说明 `11010` 当前未被占用。

---

## 4. 选择部署方式

这次实际采用的是：

- **直接下载官方 EasyTier Linux 发布包**
- **不在服务器上本地编译 Rust**
- **用 systemd 托管 `easytier-core`**

这样做的原因：

- 服务器上不一定装有 `cargo` / `rustc`
- 发布包部署更快、更稳定
- 后续升级和回滚更方便

---

## 5. 下载并安装 EasyTier

### Step 4：安装 `unzip`

```bash
apt-get update
apt-get install -y unzip
```

### Step 5：下载官方发布包

这次实际使用的版本是 `v2.4.5`。

```bash
cd /tmp
curl -L --fail -o easytier-linux-x86_64-v2.4.5.zip \
  https://github.com/EasyTier/EasyTier/releases/download/v2.4.5/easytier-linux-x86_64-v2.4.5.zip
```

如果你想确认最新版，可先在本地或服务器执行：

```bash
curl -I https://github.com/EasyTier/EasyTier/releases/latest
```

如果返回中出现类似：

```text
location: https://github.com/EasyTier/EasyTier/releases/tag/v2.4.5
```

就说明当前最新版是 `v2.4.5`。

### Step 6：解压到安装目录

```bash
rm -rf /opt/easytier/v2.4.5
mkdir -p /opt/easytier/v2.4.5
unzip -oq easytier-linux-x86_64-v2.4.5.zip -d /opt/easytier/v2.4.5
```

### Step 7：把二进制安装到系统路径

发布包内部目录结构可能会有一层子目录，因此这次实际用了兼容写法：

```bash
install -m 755 /opt/easytier/v2.4.5/easytier-core /usr/local/bin/easytier-core 2>/dev/null \
  || install -m 755 /opt/easytier/v2.4.5/*/easytier-core /usr/local/bin/easytier-core

install -m 755 /opt/easytier/v2.4.5/easytier-cli /usr/local/bin/easytier-cli 2>/dev/null \
  || install -m 755 /opt/easytier/v2.4.5/*/easytier-cli /usr/local/bin/easytier-cli
```

### Step 8：确认二进制可执行

```bash
easytier-core --help
```

如果能正常输出帮助信息，说明安装成功。

---

## 6. 创建 systemd 服务

### Step 9：创建日志目录

```bash
mkdir -p /var/log/easytier
```

### Step 10：写入 systemd 服务文件

创建文件：`/etc/systemd/system/easytier-public.service`

内容如下：

```ini
[Unit]
Description=EasyTier public relay node
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/easytier-core --hostname 你的服务器IP --instance-name astral-public --no-tun --listeners tcp://0.0.0.0:11010 udp://0.0.0.0:11010 --console-log-level info --file-log-level info --file-log-dir /var/log/easytier
Restart=always
RestartSec=5
LimitNOFILE=1048576

[Install]
WantedBy=multi-user.target
```

> 如果你要完全复现我这次的部署，把 `你的服务器IP` 换成实际公网 IP，例如：`47.119.136.200`。

### Step 11：为什么这样配置

这几个参数很关键：

- `--hostname 你的服务器IP`
  - 给节点一个容易识别的主机名
- `--instance-name astral-public`
  - 指定实例名，便于日志和状态识别
- `--no-tun`
  - 这个节点只做公网入口/中继，不在服务器本机创建 TUN 设备
- `--listeners tcp://0.0.0.0:11010 udp://0.0.0.0:11010`
  - 同时监听 TCP 和 UDP 11010
- `--console-log-level info`
  - 让 `journalctl` 里能看到关键信息
- `--file-log-level info --file-log-dir /var/log/easytier`
  - 将日志落到文件目录，方便后续排障
- `Restart=always`
  - 进程异常退出后自动重启

## 7. 启动服务

### Step 12：加载 systemd 配置并启动

```bash
systemctl daemon-reload
systemctl enable --now easytier-public.service
```

### Step 13：查看服务状态

```bash
systemctl --no-pager --full status easytier-public.service
```

你应该看到类似：

```text
Active: active (running)
```

---

## 8. 验证监听状态

### Step 14：检查端口监听

```bash
ss -lntup | grep 11010
```

预期能看到类似输出：

```text
udp   UNCONN 0      0      0.0.0.0:11010      0.0.0.0:*
tcp   LISTEN 0      1024   0.0.0.0:11010      0.0.0.0:*
```

如果还有 IPv6，也可能看到：

```text
udp   UNCONN 0      0         [::]:11010         [::]:*
tcp   LISTEN 0      1024      [::]:11010         [::]:*
```

### Step 15：从外部机器测试 TCP 可达性

在另一台机器上执行：

```bash
timeout 8 bash -lc 'cat < /dev/null > /dev/tcp/你的服务器IP/11010' && echo ok || echo fail
```

例如：

```bash
timeout 8 bash -lc 'cat < /dev/null > /dev/tcp/47.119.136.200/11010' && echo ok || echo fail
```

输出 `ok` 表示 TCP 层面可达。

## 9. 在 Astral 中使用这个服务器

### Step 16：在 Astral 客户端中添加服务器

在 UI 中添加：

- 地址：`你的服务器IP:11010`
- 勾选：`tcp`
- 勾选：`udp`

例如：

- `47.119.136.200:11010`

Astral 现有逻辑会把它展开成：

- `tcp://47.119.136.200:11010`
- `udp://47.119.136.200:11010`

### Step 17：进入同一个房间测试

要真正联机成功，除了能连到服务器，还必须：

- 房主和加入者使用同一个房间名
- 房主和加入者使用同一个房间密码
- 双方都配置了同一个公网节点或从房间分享中导入相同服务器

## 10. 查看是否有客户端连上来

### Step 18：查看当前活跃连接

```bash
ss -ntup | grep 11010 || true
```

如果有客户端连上，会看到类似：

```text
tcp   ESTAB 0 0 服务器内网IP:11010 客户端公网IP:随机端口 users:(("easytier-core",pid=...,fd=...))
```

### Step 19：查看最近日志

```bash
journalctl -u easytier-public.service --since "-30 min" --no-pager -n 300
```

如果有客户端成功进入 EasyTier 连接流程，你通常会看到类似日志：

```text
recv packet in foreign network manager
recv rpc packet in foreign network manager rpc transport
start ospf sync session
```

这些日志说明客户端不是仅仅探测端口，而是已经进入协议交互阶段。

## 12. 升级步骤

如果后续要升级到新版本，可按下面流程执行。

### Step 20：停止服务

```bash
systemctl stop easytier-public.service
```

### Step 21：下载新版并覆盖安装

以新版本号替换即可，例如：

```bash
cd /tmp
curl -L --fail -o easytier-linux-x86_64-新版本.zip \
  https://github.com/EasyTier/EasyTier/releases/download/新版本/easytier-linux-x86_64-新版本.zip

rm -rf /opt/easytier/新版本
mkdir -p /opt/easytier/新版本
unzip -oq easytier-linux-x86_64-新版本.zip -d /opt/easytier/新版本

install -m 755 /opt/easytier/新版本/easytier-core /usr/local/bin/easytier-core 2>/dev/null \
  || install -m 755 /opt/easytier/新版本/*/easytier-core /usr/local/bin/easytier-core

install -m 755 /opt/easytier/新版本/easytier-cli /usr/local/bin/easytier-cli 2>/dev/null \
  || install -m 755 /opt/easytier/新版本/*/easytier-cli /usr/local/bin/easytier-cli
```

### Step 22：重新启动并验证

```bash
systemctl start easytier-public.service
systemctl status easytier-public.service
ss -lntup | grep 11010
```

## 13. 常见问题

### 问题 1：服务是运行了，但客户端连不上

优先检查：

```bash
ss -lntup | grep 11010
systemctl status easytier-public.service
```

然后检查：

- 云服务器安全组是否放行 `11010/tcp`
- 云服务器安全组是否放行 `11010/udp`
- 本机防火墙是否拦截

### 问题 2：Astral 显示连上服务器，但房间里互通失败

这通常不是“服务器没部署好”，而是以下原因之一：

- 房间名不一致
- 房间密码不一致
- 双方没有进入同一虚拟网络
- 一端实际没有成功加入房间
- NAT / 路由 / 中继链路还在协商

### 问题 3：服务器上没有看到连接记录

用下面两条命令排查：

```bash
ss -ntup | grep 11010 || true
journalctl -u easytier-public.service --since "-30 min" --no-pager -n 300
```

如果完全没有连接痕迹，通常说明：

- 客户端没有真正发起连接
- 地址填错
- 端口未开放
- TCP/UDP 协议没有勾对

## 15. 建议

第一阶段建议先保持最小可用配置：

- 只开 `11010`
- 只跑 `TCP + UDP`
- 不启用 TUN
- 不在服务器侧额外叠加复杂配置

等你确认：

- Astral 客户端可以稳定连接
- 房间分享正常
- 多台设备能正常互联

再考虑下一步：

- 自定义更多端口
- 增加 `ws` / `wss`
- 增加反向代理或域名
- 做更复杂的完全自托管网络方案
